Git & Github Worksheet

This worksheet can be used as you work on your project to reference Terminal, Git, and Github commands

Need to know

What is Git ?

Git is a version-control system for tracking changes in computer files and coordinating work on those files among multiple people. It is primarily used for source-code management in software development, but it can be used to keep track of changes in any set of files.

What are three types of version-control systems?

Local,
Centralized,
Distributed

What is Github

GitHub Inc. is a web-based hosting service for version control using Git. It is mostly used for computer code. It offers all of the distributed version control and source code management functionality of Git as well as adding its own features

How to configure User name and Email

git config --global user.name Your Name

git config --global user.email Your Email

Basic Terminal Commands

cd

Change directory

cd ..

Go to the previous directory

mkdir

Create directory

touch

Create file

ls

List items in the directory

ls -a

List items in the directory including hidden files

clear

Clear terminal

rm

Erase file

rm

Erase file

rm -rf

Erase directory

VIM Commands

vim file.txt

To edit file

i

To begin editing

To exit VIM editor press esc and:

:wq

To save and exit

:q!

To exit and not save

Command to initialize repository

git init

To initiaze the git repository in the current folder

git init repo name

To initiaze the git repository in the specified folder

git status

To display state of working directory and staging area

Staging Changes

git add file name

To add files from the working directory to the staging area

git add -n file name

Dry run (does not unstage)

git add -A

To add all untracked files to the staging area

git rm --cached file name

To unstage (return to working directory from staging area)

git rm -f file name

To unstage files and delete

Commiting Staged Files

git commit -m "message"

To commit staged files to the repository

git commit --amend

To amend previous commit (combine current commit with previous one

git commit --amend -m “new message”

To amend previous commit with new message, replacing the old one

Annotating Commits with Tags

git tag 0.5

To annotate last commit

git tag -a 0.5 -m ‘Version estable’

Annotates last commit by giving it 0.5 tag number and a message

git tag -a 0.5sha-1

To add tag to a specific commit via the sha-1 number

git tag -l

Show list of tags

git tag -d 0.3

To delete 0.3 tag

Log Command

git log

To display all commits in repository

git log --oneline

To display all commits on oneline

git log --graph

To display all commits with a graph
